### Разберем данный код:

```yaml
#docker-compose.yaml
services:
  todoapp-postgres:
    image: postgres:18.1-bookworm
    container_name: todoapp-env-postgres
    environment:
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - POSTGRES_DB=${POSTGRES_DB}
    volumes:
      - ${PROJECT_ROOT}/out/pgdata:/var/lib/postgresql
    restart: always
```
Построчный разбор 

### Секция services
```services``` — основной раздел docker-compose, в котором описываются контейнеры.

Service — это описание контейнера, которое включает:

* образ  
* переменные окружения  
* тома  
* порты  
* сети  
* перезапуск  
* зависимости   
#### Простыми словами

```services``` = список контейнеров проекта. 

#### Пример   
```yaml
services:
  service1:
  service2:
  service3:
```  
Каждый сервис = ***отдельный*** контейнер.
### Имя сервиса

```yaml
todoapp-postgres:
```  
#### Это НЕ имя контейнера
> Имя сервиса не равно имени контейнера.  
> Имя контейнера задаётся отдельно.

### Образ контейнера
```yaml
image: postgres:18.1-bookworm
```
Это **инструкция** Docker использовать готовый образ PostgreSQL.  
Прото написанна версия которая используется
### container_name
```yaml
container_name: todoapp-env-postgres
```
Это **явное** имя контейнера
### environment
```yaml
environment:
```
Это переменные окружения контейнера  
##### Переменные окружения состоят из  
> __Ключ__=__Значение__

### Переменные окружения
```yaml 
- POSTGRES_USER=${POSTGRES_USER}
```
Задает имя пользователя базы данных  
```${POSTGRES_USER}``` - Это подстановка переменной из окружения или .env файла.  
##### В .env бедет записанно так:
> POSTGRES_USER=admin
---
```yaml
- POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
```
 Пароль для пользователя postgres.

---
```yaml
- POSTGRES_DB=${POSTGRES_DB}
```
Имя базы данных, которая создаётся автоматически.
### volumes
```yaml
volumes:
```
Секция томов (volumes).  
Volumes используются для:  

* сохранения данных
* хранения файлов вне контейнера
* persistence

##### Почему это важно
Если удалить контейнер - данные исчезнут.  
```Volumes``` - сохраняют данные на хосте.

### Маппинг тома
```yaml
- ${PROJECT_ROOT}/out/pgdata:/var/lib/postgresql
```
говорит куда сохронять данные, путь не относительный
### restart policy
```yaml
restart: always
```
Политика перезапуска контейнера.
#### always

Контейнер будет перезапускаться:  
* при падении
* при reboot системы
* при docker restart

# Makefile
Нужен для выставления переменных окружения и обращения к конкретному сервису

### Экспорт переменных из ```.env```
```Makefile
include .env
export
```

### Задаем путь для PROJECT_ROOT
```Makefile
export PROJECT_ROOT=${shell pwd}
```
Где: 
* PROJECT_ROOT - переменная котоРОЙ мы задаем параметр
* shell - выполнить в shrll
* pwd - команда которая показывает путь

### Команды Mskefile
```Makefile
env-up:
	docker compose up todoapp-postgres

env-down:
	docker compose down todoapp-postgres
```

* Первая для запуска контейнера
* Вторая для выключения

```Makefile
env-cleanup:
	@read -p "Очистить все volume файлы окружения? Опасность утери данных! [Y/N]: " ans; \
	if [ "$$ans" = "Y" ]; then \
		docker compose down todoapp-postgres && \
		sudo rm -rf out/pgdata && \
		echo "Файлы окружения очищены"; \
	else \
		echo "Очистка окружения отменена"; \
	fi
```
```@read -p "Очистить все volume файлы окружения? Опасность утери данных! [Y/N]: " ans; \``` - выводим текст и записываем переменную, @ - означает что бы не выводить исполняемую команду 
```if [ "$$ans" = "Y" ]; then \``` - если переменная = Y  тогда  
```docker compose down todoapp-postgres && \```Удаляем контейнер и  
```sudo rm -rf out/pgdata && \``` - очищаем файлы сохранения и  
```echo "Файлы окружения очищены"; \``` - выводи текст  
```else \``` - иначе  
```echo "Очистка окружения отменена"; \``` - выводим текст
```fi``` - окончание условия
